# 復古零食店 — 技術文檔

## 一、技術總覽

| 層級 | 技術 | 用途 |
|------|------|------|
| **前端** | HTML + CSS + JavaScript (Vanilla) | 純靜態網頁，無框架 |
| **前端套件** | SweetAlert2 (CDN) | 彈出提示視窗（已載入但尚未大量使用） |
| **後端** | Node.js + Express 5 | HTTP API 伺服器 |
| **資料庫** | MongoDB + Mongoose 9 (ODM) | 儲存商品、會員資料 |
| **密碼** | bcryptjs | 密碼雜湊儲存 |
| **寄信** | Resend API | 寄送註冊驗證碼 Email |
| **開發工具** | nodemon | 開發時自動重啟伺服器 |
| **容器化** | Docker Compose (MongoDB + App) | 本地開發用 |
| **部署前端** | GitHub Actions → GitHub Pages | 自動部署靜態檔案 |
| **部署後端** | Render (手動佈署) | Node.js API Server |
| **資料庫部署** | MongoDB Atlas | 正式環境用 |

## 三、前端程式碼說明

### 3.1 `index.html` — 首頁結構

**負責畫面區塊：**
- **Header**：漢堡選單（分類篩選）、Logo、搜尋欄、登入/註冊按鈕、購物車圖示
- **Banner**：三張輪播圖片
- **商品區**：由 JavaScript 動態產生商品卡片
- **彈窗（Popup）**：點擊「商品介紹」後顯示，可選擇口味、調整數量、加入購物車
- **購物車 Modal**：列出已加入的商品、總金額、刪除、結帳
- **登入/註冊 Modal**：會員登入與註冊表單，支援 Email 驗證

### 3.2 `style.css` — 全部樣式

- **主題風格**：咖啡色系（`#3b2f2f`, `#6b3e2e`, `#d4a373`），復古溫暖感
- **RWD**：使用 CSS Grid 的 `auto-fit` 搭配 `minmax`，自動適應螢幕寬度
- **重點選擇器**：
  - `.card` — 商品卡片，圓角 + hover 放大效果
  - `.popup` / `.modal` — 彈跳視窗，固定定位 + 半透明黑色背景
  - `.dropdown.show` — 用 JavaScript 控制顯示的漢堡選單
  - `input[type=number]` — 隱藏數字框的上下箭頭

### 3.3 `script.js` — 前端主要邏輯（約 309 行）

| 段落 | 函式 | 說明 |
|------|------|------|
| **1. 初始化** | `window.onload` | 頁面載入時自動：抓商品、檢查登入狀態、更新購物車數字 |
| **2. 環境判斷** | `API_URL` | 自動切換 localhost 開發環境 vs Render 正式環境 |
| **3. Banner** | `setInterval` | 每 3 秒切換一張輪播圖片 |
| **4. 商品顯示** | `fetchProducts()` → `renderProducts()` | 串接後端 `GET /api/products`，動態生成商品卡片 HTML |
| **5. 商品介紹彈窗** | `showProduct()` | 點卡片按鈕後彈窗，顯示名稱、價格、描述、口味下拉選單、數量選擇 |
| **6. 購物車** | `addToCart()` / `openCart()` / `removeItem()` / `checkout()` | 儲存在瀏覽器 `localStorage`，未串金流 |
| **7. 分類篩選** | `filterCategory()` | 漢堡選單點擊後，顯示/隱藏對應分類的卡片 |
| **8. 搜尋** | `searchProduct()` | 根據 input 文字比對 `data-name` 屬性來過濾卡片 |
| **9. 會員系統** | `register()` → `verifyCode()` → `login()` | 串接後端 API，註冊後需 Email 驗證才能登入 |
| **10. 登入狀態** | `showUser()` | 有登入時隱藏登入/註冊按鈕，顯示頭像與購物車圖示 |

### 3.4 `profile.html` — 個人資訊頁

- 獨立頁面，從 `index.html` 點擊頭像進入
- 顯示：姓名、Email（唯讀）、生日、電話、偏好設定
- **核心函式**：
  - `fetchUserInfo()` — 串接 `GET /api/user/:username` 載入完整資料
  - `saveProfile()` — 串接 `PUT /api/user/update` 儲存生日/電話/偏好
  - `logoutFromProfile()` — 清除 `localStorage` 並跳回首頁

## 四、後端程式碼說明

### 4.1 `backend/server.js`（約 231 行）

#### 環境變數（`.env` 需自行建立）
```
MONGODB_URI=mongodb://localhost:27017/retro_shop
RESEND_API_KEY=re_xxxxxxxxxxxx
PORT=3000
```

#### 各區塊說明

| 區塊 | 行數 | 說明 |
|------|------|------|
| **Middleware** | 14-16 | `express.json()` 解析 JSON body、`cors()` 開放跨域 |
| **MongoDB 連線** | 21-48 | 連線後自動檢查資料庫是否為空，為空則匯入 9 筆初始商品 |
| **User Schema** | 55-66 | 會員資料模型：帳號、密碼（hash）、Email、驗證碼與到期時間 |
| **Product Schema** | 69-77 | 商品資料模型：名稱、價格、圖片、分類、可選口味陣列 |
| **GET /api/products** | 86-93 | 回傳全部商品 |
| **POST /api/register** | 98-138 | 註冊：檢查重複 → bcrypt 加密密碼 → 產生 6 位驗證碼 → Resend 寄信 → 存檔 |
| **POST /api/verify** | 141-171 | 驗證：比對驗證碼與時效（萬用碼 `000000` 供測試） |
| **POST /api/login** | 174-191 | 登入：比對密碼 + 檢查是否已驗證 |
| **GET /api/user/:username** | 194-210 | 取得單一會員完整資料 |
| **PUT /api/user/update** | 213-225 | 更新會員生日、電話、偏好設定 |
| **伺服器啟動** | 228-231 | listen PORT (預設 3000) |

## 五、部署架構

```
GitHub Pages (靜態前端)
    │
    │ 瀏覽器發送 fetch 請求
    │
    ▼
Render Web Service (Node.js API)
    │
    │ Mongoose 連線
    │
    ▼
MongoDB Atlas
```

**流程說明：**
1. 開發者 push 程式碼到 GitHub main 分支
2. GitHub Actions 自動將靜態檔案（`index.html`, `profile.html`, `style.css`, `script.js`, `images/`）部署到 GitHub Pages
3. 使用者瀏覽 GitHub Pages 網址 → 前端 JavaScript 發送 `fetch` 請求到 Render 上的後端 API
4. Render 上的 Express 伺服器處理請求 → 讀寫 MongoDB

## 六、本地開發流程

```bash
# 1. 啟動 MongoDB + 後端（使用 Docker Compose）
docker-compose up

# 2. 或者不用 Docker，直接跑
cd backend
npm install
npm run dev    # nodemon 模式，修改程式碼會自動重啟

# 3. 前端用 Live Server 或直接開 index.html
#    瀏覽器打開 http://localhost:5500 或 http://127.0.0.1:5500
```

> 前端 `script.js` 的 `API_URL` 會自動判斷：`localhost` 就接 `http://localhost:3000/api`，否則接 Render 網址。
