# 優化實作計畫

## 目標：提升手機體驗 + 現代化 UI

---

### Phase 1 — CSS 現代化與 RWD 修正 (style.css)

#### 1.1 Header 手機版重構

**問題**：Logo + 搜尋欄 + 登入/註冊 + 購物車全擠在一排，手機無法容納

**改法**：
- 搜尋欄在手機上改為可折疊（點擊放大鏡圖示才展開）
- 登入/註冊按鈕收進漢堡選單
- 購物車圖示固定顯示
- 斷點：`@media (max-width: 768px)`

#### 1.2 Banner RWD

**問題**：`height: 320px` 固定，手機上佔太多畫面

**改法**：
- 改用 `aspect-ratio: 3 / 1` + `height: auto`
- 手機上改為 `aspect-ratio: 4 / 3`

#### 1.3 Popup 手機適配

**問題**：Popup `width: 320px` 在小手機超出畫面

**改法**：
- 改為 `width: min(90%, 500px)`
- 內容排版手機上改為垂直排列（目前是 `display: flex` 左右並排）
- 圖片縮小為手機適當尺寸

#### 1.4 現代化視覺更新

**改法**：
- 引入 Google Fonts（如 Noto Sans TC）
- 卡片加入 soft shadow、hover 微上升動畫
- 按鈕加入漸層背景 + hover 亮度變化
- Header 加入 backdrop-filter: blur() 毛玻璃效果
- 全局 transition 統一管理

#### 1.5 商品 Grid 手機版

**問題**：手機上商品卡片還是兩欄，文字圖片都太小

**改法**：
- `@media (max-width: 480px)` 改為 `grid-template-columns: 1fr` 單欄
- 卡片圖片高度適當放大

---

### Phase 2 — 前端 JS 強化 (script.js + profile.html)

#### 2.1 alert() 全面換成 SweetAlert2

**已安裝 SweetAlert2 CDN，但完全沒用到**

**改法**：
- `alert(...)` → `Swal.fire({ title, text, icon })`
- `confirm(...)` → `Swal.fire({ showCancelButton: true })`
- 購物車加入成功 → 用 SweetAlert2 toast 提示

**影響範圍（script.js）**：
- `addToCart()` 的 alert
- `register()` 的 alert
- `verifyCode()` 的 alert
- `login()` 的 alert
- `checkout()` 的 alert
- `profile.html` 裡面的 alert

#### 2.2 載入狀態改善

**問題**：商品載入中只有一行灰色文字

**改法**：
- `fetchProducts()` 期間顯示 skeleton loading 動畫（用純 CSS 實現）
- 載入失敗時顯示「載入失敗，請重新整理」按鈕
- `profile.html` 資料載入中也加入 loading 狀態

#### 2.3 圖片錯誤處理

**問題**：圖片網址失效會破圖

**改法**：
- 動態生成 img 時加上 `onerror="this.src='images/placeholder.png'"`

---

### Phase 3 — 後端小改 (server.js)

#### 3.1 CORS 鎖定

**問題**：目前 `app.use(cors())` 開放所有來源

**改法**：
```js
app.use(cors({
  origin: process.env.CORS_ORIGIN || 'https://你的帳號.github.io'
}))
```

#### 3.2 API 錯誤訊息統一格式

**問題**：錯誤回傳格式不一致

**改法**：建立一個 error handler middleware，統一回傳 `{ success: false, message }`
